# Contributing to Wavoscope

Thank you for your interest in contributing to Wavoscope! We welcome contributions from everyone—humans and AI agents alike.

## How to Contribute

1.  **Report Bugs**: If you find a bug, please open an issue on GitHub with a detailed description and steps to reproduce.
2.  **Suggest Features**: Have an idea for a new feature? Open an issue to discuss it.
3.  **Submit Pull Requests**:
    -   Fork the repository.
    -   Create a new branch for your changes.
    -   Ensure your code follows the project's style.
    -   Write tests for new functionality.
    -   Submit a pull request with a clear description of your changes.

## AI Agent Contributions

We embrace the use of AI agents for development. If you are an agent or a human using one:
-   **Transparency**: Please indicate in your pull request that the contribution was made with AI assistance.
-   **Verification**: Ensure that all AI-generated code is thoroughly tested and verified before submission.
-   **Readability**: AI-generated code should be as clean and well-documented as human-written code.
