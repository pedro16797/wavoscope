# 歌词转录与对齐

Wavoscope 提供专门的歌词轨道，用于交互式转录、词级对齐和 MusicXML 导出。此功能旨在实现快速录入和精确的时间调整。

## 核心概念

### 歌词轨道
歌词轨道是位于主波形上方的专门 Canvas 轨道。它将歌词元素显示为可编辑的方框。
- **可见性：** 使用波形顶部的“歌词”按钮切换轨道。
- **缩放：** 轨道高度固定 (32px)，但其内容随全局缩放和平移而缩放。
- **选择：** 一次只能选择一个歌词。选中歌词会高亮显示并启用特定的快捷键。

### 转录工作流：“输入-分割-前进”
转录歌曲最有效的方法是遵循以下工作流：
1.  **开始：** 按 `V` 或点击空白处，在当前播放头位置创建第一个歌词。
2.  **输入：** 输入第一个词。
3.  **分割：** 按 `空格`（用于新词）或 `-`（用于词内音节）。这将提交当前文本并立即创建一个新的歌词框。
    - **视觉区分：** 用连字符（音节）分隔的单词在时间轴上通过水平线视觉连接。连字符本身存储在数据中，但在不编辑时隐藏在 UI 中，使界面保持整洁。
4.  **重复：** 焦点自动转移到新框，允许您在音乐播放时继续转录。
5.  **完成：** 按 `回车` 或点击其他位置以结束编辑。

## 交互与快捷键

### 鼠标控制
- **单击 (空白处)：** 在该时间点添加新歌词并进入编辑模式。
- **左键单击 (现有框)：** 选中歌词。
- **单击 (已选中框)：** 取消选中歌词。
- **拖动 (中心 80%)：** 沿时间轴移动歌词元素。
- **拖动 (边缘 10%)：** 调整歌词长度。拖动左边缘调整开始时间；拖动右边缘调整持续时间。
- **双击：** 进入该歌词的编辑模式。
- **点击背景：** 取消选中当前歌词。

### 键盘快捷键

| 按键 | 动作 | 上下文 |
|-------|--------|----------|
| `V` | 添加 / 提交并前进 | 全局 |
| `Shift + V` | 取消全选 | 全局 |
| `Tab` | 切换循环模式 | 全局 |
| `回车` | 开始/结束编辑 | 选中时 |
| `Esc` | 取消编辑 / 取消选择 | 选中时 |
| `方向键 (左/右)` | 调整位置 (0.1秒) | 选中 (未编辑) |
| `方向键 (上/下)` | 调整时长 (0.1秒) | 选中 (未编辑) |
| `Shift + 方向键` | 在歌词间跳转 | 全局 / 选中时 |
| `空格 / -` | 提交并生成下一个 | 编辑中 |

## 技术实现

### 前端
- **`LyricsTimeline.tsx`**: 使用基于 Ref 的状态管理系统，在提交到后端之前在本地处理高频交互（拖动）。使用 `ResizeObserver` 实现响应式 Canvas 渲染。
- **状态管理**：集成在 `projectSlice.ts` 中。CRUD 操作经过优化，直接从后端响应更新本地状态，最大限度减少网络往返。
- **可视化**：具有动态渲染引擎，可处理小元素的文本截断和淡出。

### 后端
- **数据结构**：歌词在 `.oscope` 文件中存储为有序的 `{text, timestamp, duration}` 对象列表。
- **循环引擎**：`LoopingEngine` 支持 `lyric` 模式，该模式会自动将循环范围设置为当前选中的歌词。
- **MusicXML 导出**：`src/session/export.py` 在每个歌词和和声边界处将小节拆分为片段。这确保了 `<lyric>` 标签与导出乐谱中的节奏结构完美对齐。

## 对齐建议
- **高缩放级别：** 对于词级对齐，放大直到您可以清晰地看到波形中的瞬态。
- **循环：** 使用 `lyric` 循环模式（按 `Tab` 切换），在调整开始和结束点时重复播放当前的词。
- **节拍器：** 保持细分点击音开启，以确保您的歌词与底层的节拍标记（节奏标记）对齐。
