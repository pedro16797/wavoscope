# Структура проєкту

Цей документ описує структуру директорій та призначення кожного компонента в репозиторії Wavoscope.

## Огляд директорій

-   **`src/`**: Містить основну логіку програми та вихідний код.
    -   **`audio/`**: Містить основний аудіорушій.
        -   `audio_backend.py`: Головний фасад аудіобекенда.
        -   `chord_analyzer.py`: Детекція акордів на основі Chroma для акордних міток.
        -   `ringbuffer.py`: Потокобезпечний кільцевий буфер для потокового аудіо.
        -   `spectrum_analyzer.py`: Логіка обчислення FFT та спектральних даних.
        -   `synth.py`: Синтез у реальному часі для кліків метронома та прослуховування акордів.
        -   `waveform_cache.py`: Управління генерацією та кешуванням даних хвильової форми для ефективного відображення.
    -   **`backend/`**: Сучасний веббекенд на базі FastAPI.
        -   `main.py`: Точка входу для сервера FastAPI, обслуговує API та фронтенд-активи.
        -   `state.py`: Спільний глобальний стан (активний екземпляр `Project`).
        -   `routers/`: Модульні роутери FastAPI для різних доменів API (аудіо, відтворення, проєкт тощо).
    -   **`cli/`**: Містить утиліти командного рядка.
        -   `flag_cli.py`: Утиліти для управління мітками через термінал.
        -   `gui.py`: Логіка `pywebview` для графічного інтерфейсу користувача.
        -   `launcher.py`: Допоміжна логіка для запуску бекенда та фронтенда.
    -   **`frontend/`**: Графічний інтерфейс користувача на базі React.
        -   `src/components/`: Компоненти React (Waveform, Spectrum, Timeline, PlaybackBar).
        -   `src/store/`: Управління станом фронтенда (Zustand).
        -   `dist/`: Зібрані активи для продакшену.
        -   `tests/`: Юніт-тести для компонентів фронтенда та логіки стору.
    -   **`scripts/`**: Скрипти автоматизації та утиліти (наприклад, генерація скріншотів, створення лаунчера).
    -   **`session/`**: Обробляє збереження проєкту та високорівневий стан.
        -   `project.py`: Клас `Project`, який об'єднує аудіо, метадані (мітки) та кешування.
        -   `manager.py`: Обробляє введення/виведення sidecar-файлів `.oscope`.
        -   `flags.py`: Управління списками ритмічних та акордних міток.
        -   `undo.py`: Управління історією скасування на основі дельт.
    -   **`tests/`**: Юніт- та інтеграційні тести для бекенда та основної логіки.
    -   **`utils/`**: Загальні допоміжні функції та спільні утиліти (логірування, конфігурація тощо).
    -   `main.py`: Головна точка входу для програми.
    -   `requirements.txt`: Залежності Python.

-   **`config/`**: Конфігураційні файли та налаштування за замовчуванням для програми.
-   **`docs/`**: Документація проєкту, включаючи дорожні карти та посібники зі структури.
-   **`resources/`**: Статичні активи, такі як іконки (SVG), теми (JSON) та ресурси програми.

## Кореневі файли

-   **`run.sh` / `run.bat`**: Скрипти для налаштування середовища та запуску програми.
-   **`Wavoscope` / `Wavoscope.exe`**: Виконуваний файл лаунчера (створений скриптами).
-   **`AGENTS.md`**: Посібник та дорожня карта для AI-агентів, що працюють над проєктом.
-   **`CONTRIBUTING.md`**: Рекомендації щодо внеску в проєкт.
-   **`Readme.md`**: Загальний огляд проєкту та інструкції з налаштування.
-   **`LICENSE`**: Умови ліцензії MIT проєкту.
-   **`SECURITY.md`**: Політика повідомлення про вразливості безпеки.
