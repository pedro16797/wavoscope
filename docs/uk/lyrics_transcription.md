# Транскрипція та вирівнювання тексту

Wavoscope надає виділену доріжку тексту для інтерактивної транскрипції, вирівнювання на рівні слів та експорту в MusicXML. Ця функція розроблена для швидкого введення та точного налаштування таймінгу.

## Основні концепції

### Доріжка тексту
Доріжка тексту — це спеціалізована доріжка на основі полотна (canvas), розташована над основною хвилею. Вона відображає елементи тексту як блоки, що можна редагувати.
- **Видимість:** Перемикайте доріжку за допомогою кнопки "Lyrics" у заголовку хвилі.
- **Масштабування:** Висота доріжки фіксована (32px), але її вміст масштабується разом із глобальним зумом та зміщенням.
- **Вибір:** Одночасно може бути вибраний лише один елемент тексту. Вибір тексту підсвічує його та активує специфічні гарячі клавіші.

### Процес транскрипції: "Тип-Розділ-Перехід"
Найефективніший спосіб транскрибувати пісню — використовувати наступний алгоритм:
1.  **Початок:** Натисніть `V` або клікніть на порожньому місці, щоб створити перший фрагмент тексту в поточній позиції курсору.
2.  **Введення:** Введіть перше слово.
3.  **Розділення:** Натисніть **Пробіл** (для нових слів) або **Дефіс** (`-`, для складів у межах слова). Це підтверджує поточний текст і негайно створює нове поле для тексту.
    - **Візуальна диференціація:** Слова, розділені дефісами (склади), візуально з'єднуються горизонтальною лінією на часовій шкалі. Сам дефіс зберігається в даних, але приховується в інтерфейсі, коли текст не редагується, що забезпечує охайний вигляд.
4.  **Повторення:** Фокус автоматично переноситься на нове поле, що дозволяє продовжувати транскрипцію під час відтворення музики.
5.  **Підтвердження:** Натисніть `Enter` або клікніть в іншому місці, щоб завершити редагування.

## Взаємодія та гарячі клавіші

### Керування мишею
- **Один клік (порожнє місце):** Додає новий фрагмент тексту в цій часовій точці та входить у режим редагування.
- **Лівий клік (існуючий блок):** Вибирає фрагмент тексту.
- **Один клік (вибраний блок):** Знімає виділення.
- **Перетягування (центральні 80%):** Переміщує елемент тексту вздовж часової шкали.
- **Перетягування (краї 10%):** Змінює розмір елемента. Перетягування лівого краю регулює час початку; перетягування правого краю регулює тривалість.
- **Подвійний клік:** Входить у режим редагування вибраного фрагмента.
- **Клік по фону:** Знімає виділення з поточного фрагмента.

### Гарячі клавіші

| Клавіша | Дія | Контекст |
|-----|--------|---------|
| `V` | Додати / Підтвердити та перейти | Глобально |
| `Shift + V` | Зняти все виділення | Глобально |
| `Tab` | Перемкнути режим петлі | Глобально |
| `Enter` | Почати/Завершити редагування | Вибрано |
| `Escape` | Скасувати редагування / Зняти виділення | Вибрано |
| `Стрілки / `A` / `D`` | Змістити позицію (0.1с) | Вибрано (не редагується) |
| `Стрілки / `W` / `S`` | Налаштувати тривалість (0.1с) | Вибрано (не редагується) |
| `Shift + Стрілки` | Перехід між словами | Глобально / Вибрано |
| `Пробіл / -` | Підтвердити та створити наступне | Редагування |

## Технічна реалізація

### Frontend
- **`LyricsTimeline.tsx`**: Використовує систему управління станом на основі Ref для локальної обробки високоефективних взаємодій (перетягування) перед відправкою на бекенд. Використовує `ResizeObserver` для адаптивного рендерингу полотна.
- **Управління станом**: Інтегровано в `projectSlice.ts`. Операції CRUD оптимізовані для оновлення локального стану безпосередньо з відповідей бекенда, що мінімізує кількість мережевих запитів.
- **Візуалізація**: Має динамічний рушій рендерингу, який обробляє обрізання та згасання тексту для малих елементів.

### Backend
- **Структура даних**: Текст зберігається як відсортований список об'єктів `{text, timestamp, duration}` у sidecar-файлі `.oscope`.
- **Рушій петлі**: `LoopingEngine` підтримує режим `lyric`, який автоматично встановлює діапазон петлі на вибране слово тексту.
- **Експорт MusicXML**: `src/session/export.py` розділяє такти на сегменти на кожній межі тексту та акорду. Це гарантує, що теги `<lyric>` ідеально вирівняні з ритмічною структурою в експортованій партитурі.

## Поради щодо вирівнювання
- **Високий зум:** Для вирівнювання на рівні слів збільште масштаб, поки не побачите чіткі транзієнти на хвильовій формі.
- **Петля:** Використовуйте режим петлі `lyric` (перемикання клавішею `Tab`), щоб повторювати поточне слово під час точного налаштування його початку та кінця.
- **Метроном:** Тримайте кліки поділок увімкненими, щоб переконатися, що ваш текст збігається з основними маркерами біта (Ритмічними мітками).
