# 가사 전사 및 정렬

Wavoscope는 대화형 전사, 단어 수준 정렬 및 MusicXML 내보내기를 위한 전용 가사 트랙을 제공합니다. 이 기능은 빠른 입력과 정밀한 타이밍 조정을 위해 설계되었습니다.

## 핵심 개념

### 가사 트랙
가사 트랙은 메인 파형 위에 위치한 전용 캔버스 기반 트랙입니다. 가사 요소를 편집 가능한 상자로 표시합니다.
- **가시성:** 파형 헤더의 "Lyrics" 버튼을 사용하여 트랙을 토글합니다.
- **크기 조정:** 트랙 높이는 고정(32px)되어 있지만, 그 내용은 글로벌 확대/축소 및 오프셋에 따라 크기가 조정됩니다.
- **선택:** 한 번에 하나의 가사만 선택할 수 있습니다. 가사를 선택하면 강조 표시되며 특정 키보드 단축키가 활성화됩니다.

### 전사 워크플로우: "입력-분할-전진"
가장 효율적인 전사 방법은 다음 워크플로우를 사용하는 것입니다:
1.  **시작:** `V`를 누르거나 빈 공간을 클릭하여 현재 재생 헤드 위치에 첫 번째 가사를 생성합니다.
2.  **입력:** 첫 번째 단어를 입력합니다.
3.  **분할:** **스페이스**(새 단어의 경우) 또는 **대시**(`-`, 단어 내 음절의 경우)를 누릅니다. 그러면 현재 텍스트가 확정되고 즉시 새 가사 상자가 생성됩니다.
    - **시각적 차별화:** 하이픈으로 분할된 단어(음절)는 타임라인에서 가로선으로 시각적으로 연결됩니다. 하이픈 자체는 데이터에 저장되지만 편집 중이 아닐 때는 UI에서 숨겨져 깔끔하게 보입니다.
4.  **반복:** 포커스가 자동으로 새 상자로 이동하여 음악이 재생되는 동안 전사를 계속할 수 있습니다.
5.  **확정:** `Enter`를 누르거나 다른 곳을 클릭하여 편집을 마칩니다.

## 상호작용 및 단축키

### 마우스 제어
- **한 번 클릭 (빈 공간):** 해당 타임스탬프에 새 가사를 추가하고 편집 모드로 들어갑니다.
- **왼쪽 클릭 (기존 상자):** 가사를 선택합니다.
- **한 번 클릭 (선택된 상자):** 선택을 해제합니다.
- **드래그 (중앙 80%):** 가사 요소를 타임라인을 따라 이동합니다.
- **드래그 (가장자리 10%):** 가사 크기를 조정합니다. 왼쪽 가장자리를 드래그하면 시작 시간이 조정되고, 오른쪽 가장자리를 드래그하면 지속 시간이 조정됩니다.
- **더블 클릭:** 클릭한 가사의 편집 모드로 들어갑니다.
- **배경 클릭:** 현재 가사 선택을 해제합니다.

### 키보드 단축키

| 키 | 동작 | 컨텍스트 |
|-----|--------|---------|
| `V` | 추가 / 확정 및 전진 | 글로벌 |
| `Shift + V` | 모두 선택 해제 | 글로벌 |
| `Tab` | 루프 모드 전환 | 글로벌 |
| `Enter` | 편집 시작/종료 | 선택됨 |
| `Escape` | 편집 취소 / 선택 해제 | 선택됨 |
| `화살표 / A / D` | 위치 미세 조정 (0.1초) | 선택됨 (편집 중 아님) |
| `화살표 / W / S` | 지속 시간 조정 (0.1초) | 선택됨 (편집 중 아님) |
| `Shift + 화살표` | 가사 사이 이동 | 글로벌 / 선택됨 |
| `스페이스 / -` | 확정 및 다음 생성 | 편집 중 |

## 기술적 구현

### 프론트엔드
- **`LyricsTimeline.tsx`**: 백엔드에 확정하기 전에 로컬에서 고빈도 상호작용(드래그 등)을 처리하기 위해 Ref 기반 상태 관리 시스템을 사용합니다. 반응형 캔버스 렌더링을 위해 `ResizeObserver`를 사용합니다.
- **상태 관리**: `projectSlice.ts`에 통합되어 있습니다. CRUD 작업은 네트워크 왕복을 최소화하기 위해 백엔드 응답에서 직접 로컬 상태를 업데이트하도록 최적화되어 있습니다.
- **시각적 요소**: 작은 요소에 대해 텍스트 생략 및 페이딩을 처리하는 동적 렌더링 엔진을 갖추고 있습니다.

### 백엔
- **데이터 구조**: 가사는 `.oscope` 사이드카 파일 내에 `{text, timestamp, duration}` 개체의 정렬된 목록으로 저장됩니다.
- **루핑 엔진**: `LoopingEngine`은 `lyric` 모드를 지원하며, 현재 선택된 가사에 맞게 루프 범위를 자동으로 설정합니다.
- **MusicXML 내보내기**: `session/export.py`는 모든 가사와 코드 경계에서 마디를 세그먼트로 나눕니다. 이를 통해 내보낸 악보에서 `<lyric>` 태그가 리듬 구조와 완벽하게 일치하도록 보장합니다.

## 정렬 팁
- **고배율 확대:** 단어 수준 정렬을 위해 파형의 트랜지언트(음의 시작 부분)가 명확하게 보일 때까지 확대합니다.
- **루핑:** `lyric` 루프 모드(`Tab`으로 전환)를 사용하여 시작 및 종료 지점을 미세 조정하는 동안 현재 단어를 반복합니다.
- **메트로놈:** 세분화 클릭을 켜서 가사가 기본 비트 마커(리듬 플래그)와 일치하는지 확인합니다.
