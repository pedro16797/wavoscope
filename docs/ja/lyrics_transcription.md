# 歌詞の文字起こしと配置

Wavoscopeは、インタラクティブな文字起こし、単語レベルの配置、およびMusicXMLエクスポートのための専用の歌詞トラックを提供します。この機能は、迅速な入力と正確なタイミング調整のために設計されています。

## 基本コンセプト

### 歌詞トラック
歌詞トラックは、メイン波形の上に配置された専用のキャンバスベースのトラックです。歌詞要素を編集可能なボックスとして表示します。
- **表示:** 波形ヘッダーの「歌詞」ボタンを使用してトラックを切り替えます。
- **スケーリング:** トラックの高さは固定（32px）ですが、その内容はグローバルなズームとオフセットに合わせてスケーリングされます。
- **選択:** 一度に選択できる歌詞は1つだけです。歌詞を選択するとハイライトされ、特定のキーボードショートカットが有効になります。

### 文字起こしワークフロー: 「入力・分割・前進」
最も効率的な文字起こしの方法は、以下のワークフローを使用することです：
1.  **開始:** `V`を押すか空きスペースをクリックして、現在の再生ヘッド位置に最初の歌詞を作成します。
2.  **入力:** 最初の単語を入力します。
3.  **分割:** **スペース**（新しい単語の場合）または**ハイフン**（`-`、単語内の音節の場合）を押します。これにより現在のテキストが確定され、直後に新しい歌詞ボックスが生成されます。
    - **視覚的区別:** ハイフンで分割された単語（音節）は、タイムライン上で水平線によって視覚的に接続されます。ハイフン自体はデータに保存されますが、編集時以外はUI上で非表示になり、すっきりとした見た目になります。
4.  **繰り返し:** フォーカスは自動的に新しいボックスに移るため、音楽を再生しながら文字起こしを続けることができます。
5.  **確定:** `Enter`を押すか他の場所をクリックして編集を終了します。

## 操作とショートカット

### マウス操作
- **シングルクリック（空きスペース）:** そのタイムスタンプに新しい歌詞を追加し、編集モードに入ります。
- **左クリック（既存のボックス）:** 歌詞を選択します。
- **シングルクリック（選択済みのボックス）:** 選択を解除します。
- **ドラッグ（中央80%）:** 歌詞要素をタイムラインに沿って移動します。
- **ドラッグ（端の10%）:** 歌詞のサイズを変更します。左端をドラッグすると開始時間が調整され、右端をドラッグすると長さが調整されます。
- **ダブルクリック:** クリックした歌詞の編集モードに入ります。
- **背景クリック:** 現在の歌詞の選択を解除します。

### キーボードショートカット

| キー | アクション | コンテキスト |
|-----|--------|---------|
| `V` | 追加 / 確定して前進 | グローバル |
| `Shift + V` | すべて選択解除 | グローバル |
| `Tab` | ループモード切り替え | グローバル |
| `Enter` | 編集開始/終了 | 選択中 |
| `Escape` | 編集キャンセル / 選択解除 | 選択中 |
| `矢印 / A / D` | 位置を微調整 (0.1s) | 選択中 (非編集時) |
| `矢印 / W / S` | 長さを調整 (0.1s) | 選択中 (非編集時) |
| `Shift + 矢印` | 歌詞間をシーク | グローバル / 選択中 |
| `スペース / -` | 確定して次を生成 | 編集モード |

## 技術的実装

### フロントエンド
- **`LyricsTimeline.tsx`**: バックエンドへ確定する前に、高頻度の操作（ドラッグなど）をローカルで処理するためにRefベースの状態管理システムを使用しています。レスポンシブなキャンバスレンダリングのために`ResizeObserver`を採用しています。
- **状態管理**: `projectSlice.ts`に統合されています。CRUD操作は、バックエンドからのレスポンスから直接ローカル状態を更新するように最適化されており、ネットワークの往復を最小限に抑えています。
- **ビジュアル**: 小さな要素に対してテキストの切り詰めやフェードアウトを処理する動的なレンダリングエンジンを備えています。

### バックエンド
- **データ構造**: 歌詞は`.oscope`サイドカーファイル内に`{text, timestamp, duration}`オブジェクトのソート済みリストとして保存されます。
- **ループエンジン**: `LoopingEngine`は`lyric`モードをサポートしており、現在選択されている歌詞に合わせてループ範囲を自動的に設定します。
- **MusicXMLエクスポート**: `session/export.py`は、すべての歌詞とコードの境界で小節をセグメントに分割します。これにより、エクスポートされたスコア内で`<lyric>`タグがリズム構造と完全に一致することが保証されます。

## 配置のコツ
- **高倍率ズーム:** 単語レベルの配置を行う際は、波形のトランジェント（立ち上がり）がはっきりと見えるまでズームインしてください。
- **ループ:** `lyric`ループモード（`Tab`で切り替え）を使用して、開始点と終了点を微調整しながら現在の単語を繰り返します。
- **メトロノーム:** 分割クリックをオンにして、歌詞が下層のビートマーカー（リズムフラグ）と一致していることを確認してください。
