# プロジェクト構成

このドキュメントでは、Wavoscopeリポジトリのディレクトリ構成と各コンポーネントの目的について説明します。

## ディレクトリ概要

-   **`src/`**: コアアプリケーションロジックとソースコードが含まれます。
    -   **`audio/`**: コアオーディオエンジンが含まれます。
        -   `audio_backend.py`: メインのオーディオバックエンド・ファサード。
        -   `chord_analyzer.py`: コードフラグ用のChromaベースのコード検出。
        -   `ringbuffer.py`: オーディオストリーミング用のスレッドセーフなリングバッファ。
        -   `spectrum_analyzer.py`: FFTとスペクトラムデータの計算ロジック。
        -   `synth.py`: メトロノームクリックとコード試聴用のリアルタイム合成。
        -   `waveform_cache.py`: 効率的な表示のための波形データの生成とキャッシュを管理。
    -   **`backend/`**: FastAPIベースのモダンなWebバックエンド。
        -   `main.py`: FastAPIサーバーのエントリポイント。APIエンドポイントとフロントエンドアセットを提供。
        -   `state.py`: 共有グローバル状態（アクティブな`Project`インスタンス）。
        -   `routers/`: 各APIドメイン（オーディオ、再生、プロジェクトなど）ごとのモジュール式FastAPIルーター。
    -   **`cli/`**: コマンドラインインターフェース・ユーティリティ。
        -   `flag_cli.py`: ターミナル経由でフラグを管理するためのユーティリティ。
        -   `gui.py`: グラフィカルユーザーインターフェースのための`pywebview`ロジック。
        -   `launcher.py`: バックエンドとフロントエンドを起動するためのヘルパーロジック。
    -   **`frontend/`**: Reactベースのグラフィカルユーザーインターフェース。
        -   `src/components/`: Reactコンポーネント（Waveform, Spectrum, Timeline, PlaybackBar）。
        -   `src/store/`: フロントエンドの状態管理（Zustand）。
        -   `dist/`: ビルド済みのプロダクションアセット。
        -   `tests/`: フロントエンドコンポーネントとストアロジックのユニットテスト。
    -   **`scripts/`**: 自動化およびユーティリティスクリプト（例：スクリーンショット生成、ランチャー作成）。
    -   **`session/`**: プロジェクトの永続化と高レベルの状態を処理。
        -   `project.py`: オーディオ、メタデータ（フラグ）、キャッシュを統合する`Project`クラス。
        -   `manager.py`: `.oscope`サイドカーファイルの入出力を処理。
        -   `flags.py`: リズムフラグとコードフラグのリストを管理。
        -   `undo.py`: デルタベースの元に戻す履歴管理。
    -   **`tests/`**: バックエンドとコアロジックのユニットおよび統合テスト。
    -   **`utils/`**: 汎用的なヘルパー関数と共有ユーティリティ（ロギング、構成など）。
    -   `main.py`: アプリケーションの主要なエントリポイント。
    -   `requirements.txt`: Pythonの依存関係。

-   **`config/`**: アプリケーションの構成ファイルとデフォルト設定。
-   **`docs/`**: タスクロードマップや構成ガイドを含むプロジェクトドキュメント。
-   **`resources/`**: アイコン（SVG）、テーマ（JSON）、アプリケーションリソースなどの静的アセット。

## ルートファイル

-   **`run.sh` / `run.bat`**: 環境セットアップとアプリケーション起動用のスクリプト。
-   **`Wavoscope` / `Wavoscope.exe`**: ビルド済みのランチャー実行ファイル（スクリプトにより生成）。
-   **`AGENTS.md`**: プロジェクトに従事するAIエージェント向けのガイダンスとロードマップ。
-   **`CONTRIBUTING.md`**: プロジェクトへの貢献ガイドライン。
-   **`Readme.md`**: プロジェクトの概要とセットアップ手順。
-   **`LICENSE`**: プロジェクトのMITライセンス条項。
-   **`SECURITY.md`**: セキュリティ脆弱性の報告に関するポリシー。
